## Як це працює

Кожен конспект — **один текстовий файл** у теці `notes/`. Сайт сам знаходить
усі такі файли й будує з них сторінки, меню та колоди карток.

Файл поділений на три секції — це три режими на сайті:

- **Конспект** — теорія, яку читаєш;
- **Картки** — флеш-картки: натискаєш, картка перевертається;
- **Завдання** — задачі з прихованими розв'язками.

Будь-яку секцію можна пропустити — вкладка просто не з'явиться.

Формули набрані в LaTeX і рендеряться через [KaTeX](https://katex.org).
Схеми намальовані як SVG прямо в тексті конспекту: вони чіткі на будь-якому
екрані й самі підлаштовуються під світлу й темну тему.

## Запуск

Потрібен лише Python 3 — жодних залежностей ставити не треба.

```bash
cd math-notes
python3 serve.py
```

Далі відкрити http://localhost:8765. Зупинити — `Ctrl+C`.

`serve.py` перескладає список конспектів на кожен запит і забороняє браузеру
кешувати сайт, тому зміни видно одразу після оновлення сторінки.

## Як додати конспект

1. Створити файл `notes/<розділ>/<тема>/<назва>.md`
2. Оновити сторінку в браузері

Нова тека всередині `notes/` автоматично стає новим розділом або темою.

## Формат файлу

```markdown
---
title: Види трикутників       ← назва на сайті
section: Геометрія            ← розділ
topic: Трикутники             ← тема (необов'язково; збирає підтеми разом)
order: 1                      ← порядок вивчення всередині теми
---

## Картки

### Питання на лицьовому боці

Відповідь, яка з'являється після перевертання.

### Який це трикутник?

<svg viewBox="0 0 90 85">…</svg>

---

Малюнок вище — на **питанні**: усе до рядка `---` належить лицьовому боку.

## Завдання

### Умова задачі

Розв'язок, прихований до натискання кнопки.
```

Кожне `###` усередині секції — окрема картка або окреме завдання.

**Увага:** всередині SVG не має бути порожніх рядків — Markdown на них
вважає, що HTML скінчився, і показує решту як код.

## Як працюють картки

Одна картка на екран. Натискаєш на неї — перевертається. Стрілки `←` `→`
гортають колоду, **Перемішати** тасує порядок.

Клавіші: пробіл — перевернути, `←` `→` — назад і вперед.

Сайт нічого не запам'ятовує: ні оцінок, ні прогресу. Щоразу колода
починається з чистого аркуша — це навмисно.

## Перед публікацією

На статичному хостингу (GitHub Pages, Netlify) сервера немає — там просто
лежать файли. Тому список конспектів треба скласти заздалегідь:

```bash
python3 build.py
```

Інакше сайт в інтернеті показуватиме **старий** список конспектів.

## Файли

- `notes/` — конспекти (єдине, що треба чіпати)
- `serve.py` — локальний сервер
- `build.py` — складає `notes/index.json` зі списком конспектів
- `index.html`, `src/app.js`, `src/style.css` — сам сайт

## Ліцензія

Конспекти написані власними словами на основі шкільних довідників.
Математичні факти нікому не належать; текст, схеми й код — вільні
для використання.
